# NYC-Taxi-Visualization

NYC 出租车数据可视化分析项目，基于 **Python FastAPI 后端 + Vue3 前端** 实现前后端分离，包含**完整版数据清洗**+**交互式可视化图表**，小白可一键部署运行。

## 技术栈

- **后端**：Python 3.9+、FastAPI、Uvicorn、Pandas

- **前端**：Vue3、Vite4、ECharts、Axios

- **数据**：Parquet 原始数据、CSV 区域映射表

## 项目结构

```shell
NYC-Taxi-Visualization/       
├── .vscode/         		  # VSCode 快捷任务配置
│   └── tasks.json            # 一键启动/清洗任务
├── data/                     # 原始数据目录
│   ├── green/                # Green 出租车 parquet 数据
│   ├── yellow/               # Yellow 出租车 parquet 数据
│   └── taxi_zone_lookup.csv  # 区域映射表
├── backend/         		  # Python FastAPI 后端服务
│   ├── clean_data.py         # 独立数据清洗脚本（仅运行一次）
│   └── main.py               # 后端API服务
├── frontend/        		  # Vue3 前端可视化项目
└── README.md                 # 项目说明文档
```

## 二、快捷启动方式（推荐）：VSCode Task 一键运行

项目已配置 `VSCode Task`，**无需手动切换目录、无需记忆命令**，一键执行数据清洗 / 启动前后端。

### 使用步骤

1. 确保项目根目录存在 `.vscode/tasks.json` 配置文件

2. 打开 VSCode，在上方搜索窗口输入`task`：

3. 在弹出的任务列表中选择对应功能：

    - **运行数据清洗脚本**：首次使用必执行，生成清洗后数据

    - **启动后端服务**：启动 API 接口服务

    - **启动前端服务**：启动可视化页面

### 任务说明

|任务名称|作用|执行频率|
|---|---|---|
|运行数据清洗脚本|生成清洗后的标准化数据|首次使用执行 1 次|
|启动后端服务|运行 FastAPI 数据接口|每次运行启动|
|启动前端服务|运行 Vue 可视化页面|每次运行启动|

## 三、传统启动方式（双终端手动运行）

需要**打开两个独立终端**，同时启动前后端服务

### 终端 1：执行数据清洗（首次使用必运行）

```bash
cd backend
python clean_data.py
```

### 终端 2：启动 Python 后端服务

```bash
cd backend
uvicorn main:app --reload
```

✅ 后端启动成功：`http://127.0.0.1:8000`

### 终端 3：启动 Vue 前端

```bash
cd frontend
npm run dev
```

✅ 前端启动成功：`http://localhost:5173`

## 五、数据清洗规则

本项目对 Green 出租车与 Yellow 出租车采用**两套独立官方清洗逻辑**，保证数据准确、干净、可直接用于可视化分析。

### 绿车（Green Taxi）数据清洗逻辑

1. **字段重命名**
按官方字典将原始英文字段统一转为中文，保留全部原始字段，不删除任何列。

2. **删除重复记录**
去除完全重复的行程数据，避免统计结果偏大。

3. **时间字段标准化**
将上车、下车时间转为标准日期时间格式，无法转换的数据标记为空。

4. **缺失值处理**
删除上车时间、下车时间为空的无效行程。

5. **时间范围过滤**
仅保留 2018 年的行程记录，剔除混入的其他年份数据。

6. **总费用修正（核心）**
按官方计费规则重新计算总费用，空值字段自动填充为 0，避免计算出错。
总费用 = 车费 + 额外附加费 + MTA 税费 + 改善附加费 + 小费 + 通行费

7. **异常值过滤**

    - 乘客数量：1–6 人

    - 行程距离 > 0

    - 车费、总费用不为负

    - 下车时间晚于上车时间

8. **输出清洗结果**
生成字段完整、费用正确、无脏数据的清洗后文件，保存至 `data/clean_data`。

---

### 黄车（Yellow Taxi）数据清洗逻辑

1. **字段重命名**
按官方字典将英文字段转为中文，保留所有原始列，不删除不存在的空字段。

2. **删除重复记录**
去除重复行程记录，保证数据唯一性。

3. **时间字段标准化**
将上车、下车时间转为标准时间格式，异常时间转为空值。

4. **缺失值处理**
删除上车时间、下车时间为空的无效数据。

5. **时间范围过滤**
仅保留 2018 年行程，过滤错误年份数据。

6. **总费用修正（核心）**
按 2018 年官方规则重新计算总费用，不存在的费用项不参与计算。
总费用 = 车费 + 额外附加费 + MTA 税费 + 改善附加费 + 小费 + 通行费

7. **异常值过滤**

    - 乘客数量：1–6 人

    - 行程距离 > 0

    - 车费、总费用 ≥ 0

    - 下车时间晚于上车时间

8. **输出清洗结果**
输出字段完整、总费用正确、可直接用于可视化分析的数据集。

---

### 通用清洗规则

1. 关联区域映射表，剔除缺失行政区信息的无效数据

2. 提取上车小时，用于 24 小时高峰趋势分析

3. 清洗完成后自动保存至 `data/clean_data` 目录

4. 支持批量清洗全年 / 全月数据，无需手动处理
